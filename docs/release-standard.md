# Release Standard

## Purpose

Releases help document progress and provide credibility.

## Recommended format

```text
v0.1.0 — Initial public foundation
v0.2.0 — Community-ready documentation
v0.3.0 — First usable prototype
v1.0.0 — Stable release
```

## Release notes should include

- Summary
- Added
- Changed
- Fixed
- Known limitations
- Contributors
- Next steps

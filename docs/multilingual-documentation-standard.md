# Multilingual Documentation Standard

SaaSCoop adopts multilingual documentation as part of its commitment to open knowledge, accessibility and international cooperation.

## Supported languages

Institutional documentation should be available in:

- Portuguese (`pt-BR`)
- English (`en`)
- Spanish (`es`)

## Language structure

```text
docs/
├── pt-BR/
├── en/
└── es/
```

## Reference language

English is the reference language for technical interoperability.

Portuguese and Spanish have equal institutional importance and should be maintained as first-class documentation.

## Required institutional documents

Whenever possible, the following documents should exist in all three languages:

- README
- ROADMAP
- FAQ
- ECOSYSTEM
- GET_INVOLVED
- GOVERNANCE

## Translation status

Each translated document should indicate one of the following:

```text
Translation status: synced
Translation status: needs review
Translation status: draft
```

## Principle

Open source is not only about open code.

For SaaSCoop, open source also means open knowledge, accessible language and community participation.

# Documentation Standard

## Purpose

Documentation is part of the product.

Every SaaSCoop repository should be understandable by:

- technical contributors
- non-technical community members
- maintainers
- potential funders
- partner organizations

## Recommended README structure

- Project name
- Short description
- Purpose
- Status
- Features
- Getting started
- How to contribute
- Governance
- License

## Language

English is recommended for repository structure and technical interoperability.

Portuguese and Spanish are welcome for educational materials and community documentation.
